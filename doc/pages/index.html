Overview

What is TOMCAT C2

TOMCAT-C2 is a Command and Control server built in Java, engineered for authorized red team engagements. It manages multiple concurrent reverse shell sessions across diverse target environments from a single interactive operator console.

The framework is designed around operational security — encrypted TCP channels, minimal target footprint, configurable listener ports, and comprehensive session logging for post-engagement reporting.

Features

Core Capabilities

Everything a red team operator needs in a single Java-based framework.

01

Multi-Session Handler

Manage multiple simultaneous reverse shell sessions from a single console. Switch between targets without losing context.

02

Encrypted Communications

AES-256 encrypted C2 traffic prevents traffic analysis and ensures operational confidentiality between operator and agent.

03

Interactive Shell

Full interactive shell with real-time output streaming. Execute commands and receive responses as if local.

04

File Transfer

Upload payloads and download evidence files with built-in progress tracking and integrity verification.

05

Listener Management

Spin up and tear down listeners on demand. Configure multiple ports simultaneously for flexible callback options.

06

OPSEC Design

Minimal target footprint, configurable beacon intervals, and encrypted payloads engineered for stealth engagements.

07

Session Persistence

Resume sessions after server restarts. Session state is maintained across network interruptions automatically.

08

Activity Logging

Full audit trail of all commands and outputs. Timestamped logs saved to disk for post-engagement review and reporting.

09

Modular Core

Clean Java module architecture makes extending the framework with custom payloads and handlers straightforward.

Installation

Getting Started

Deploy TOMCAT C2 in under 5 minutes.

01

Prerequisites

Ensure the following are installed before proceeding.

OpenJDK 17 or newer Required
Apache Maven 3.8+ Required
Git Required
Linux (Debian recommended) Recommended
02

Clone Repository

Pull the source from GitHub.

bash 
# Clone the repository
git clone https://github.com/MatrixTM26/TOMCAT-C2-Framework.git
cd TOMCAT-C2-Framework
03

Build with Maven

Compile and package the project into an executable JAR.

bash 
# Build the JAR
mvn clean package -DskipTests

# Or use the provided script
bash build.sh
04

Configure

Edit server.properties with your listener settings.

properties 
server.host=0.0.0.0
server.port=4444
server.timeout=30000
output.dir=./output
05

Launch

Start the C2 server. The operator console initializes once the listener is active.

bash 
# Start the server
java -jar target/tomcat-c2.jar
Server running. Waiting for incoming connections.

Architecture

System Architecture

How TOMCAT C2 components interconnect in a live deployment.

Operator Console

Java interactive terminal. Full command history, tab completion, real-time session output.

C2 Server Core

Multi-threaded session handler manages listener threads, routes commands, tracks agent state.

Reverse Shell Agents

Lightweight callbacks on targets maintain encrypted TCP channels back to the server.

Output Logging

All session I/O and commands captured to timestamped log files for post-engagement review.
