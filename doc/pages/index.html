Overview

What is TOMCAT C2

A powerful post-exploitation framework engineered for red team engagements and penetration testing workflows.

TOMCAT-C2 Framework is a Command and Control (C2) server built with Java, designed to manage multiple reverse shell sessions simultaneously across heterogeneous target environments. It provides operators with a centralized control plane for post-exploitation activities during authorized red team operations.

The framework prioritizes operational security (OPSEC), encrypted communications, and extensibility, making it suitable for enterprise-grade red teaming assessments where stealth and reliability are critical requirements.

Features

Core Capabilities

Everything you need for comprehensive red team operations in a single cohesive framework.

Multi-Session Handler

Manage multiple simultaneous reverse shell sessions from a single operator console. Switch between targets seamlessly without losing context.

Encrypted Communications

All C2 traffic is encrypted in transit using cryptographic protocols, ensuring operational security and preventing traffic analysis.

Interactive Shell

Full interactive shell experience with command history, tab completion support, and real-time output streaming from remote sessions.

File Transfer

Upload and download files to and from compromised endpoints with built-in progress tracking and integrity verification.

Listener Management

Configure and manage multiple listeners on different ports and protocols. Supports TCP-based shell callbacks with flexible bind options.

OPSEC Focused

Designed with operational security in mind. Minimal footprint, configurable callback intervals, and encrypted payloads for stealth operations.

Session Persistence

Session state management allows operators to reconnect and resume control of active sessions across server restarts and network interruptions.

Activity Logging

Comprehensive audit trail of all operator commands and session activities. Output logs stored to the designated output directory for review.

Modular Architecture

Built with extensibility at its core. The Java-based module system allows operators to extend functionality with custom payloads and handlers.

Installation

Getting Started

Deploy TOMCAT C2 in minutes with the following step-by-step guide.

01

Prerequisites

Ensure the following dependencies are installed on your system before proceeding.

OpenJDK 17 or newer Required
Apache Maven 3.8+ Required
Git Required
Linux (Debian based recommended) Recommended
02

Clone Repository

Clone the TOMCAT-C2-Framework repository from GitHub to your local machine.

bash 
# Clone the repository
git clone https://github.com/MatrixTM26/TOMCAT-C2-Framework.git
cd TOMCAT-C2-Framework
03

Build with Maven

Compile and package the project using Maven. This will generate the executable JAR file.

bash 
# Build the project
mvn clean package -DskipTests

Alternatively, use the provided build script: bash build.sh

04

Configure Server

Edit server.properties to configure your listener settings and operational parameters.

properties 
server.host=0.0.0.0
server.port=4444
server.timeout=30000
output.dir=./output
05

Launch the C2 Server

Start the TOMCAT C2 server. The interactive console will launch once the listener is active.

bash 
# Start the server
java -jar target/tomcat-c2.jar
Server is running. Waiting for incoming connections on the configured port.
Commands

Command Reference

Full reference of available commands in the TOMCAT C2 interactive operator console.

Command Usage Description
help help Display all available commands and their descriptions
sessions sessions List all active shell sessions with their IDs and details
listeners listeners Show active listeners and their bind configurations
listen listen [port] Start a new listener on the specified port
stop stop [port] Stop an active listener on the specified port
clear clear Clear the operator console screen
exit exit Gracefully shut down the C2 server and all sessions
Command Usage Description
interact interact [id] Switch to an interactive shell with the specified session
kill kill [id] Terminate the specified shell session
killall killall Terminate all active sessions simultaneously
info info [id] Display detailed information about a specific session
background background Background current interactive session and return to main console
rename rename [id] [name] Assign a label to a session for easier identification
Command Usage Description
upload upload [local] [remote] Upload a local file to the remote target
download download [remote] [local] Download a file from the remote target to local machine
ls ls [path] List files and directories on the remote target
cd cd [path] Change working directory on the remote session
cat cat [file] Read and display file contents from the remote target
Command Usage Description
sysinfo sysinfo Retrieve operating system and hardware information from target
whoami whoami Display current user context on the remote session
ps ps List running processes on the remote target
screenshot screenshot Capture a screenshot of the remote target desktop
shell shell [cmd] Execute a single command on the target without entering interactive mode
Architecture

System Architecture

Understanding how TOMCAT C2 components interact in a typical deployment.

Operator Console

The Java-based interactive terminal where operators issue commands, manage sessions, and orchestrate the engagement workflow.

C2 Server Core

Central session handler that manages TCP listener threads, incoming connections, and routes operator commands to target agents.

Reverse Shell Agents

Lightweight shell callbacks deployed on target systems that maintain encrypted communication channels back to the C2 server.

Output Logging

All session output and operator commands are captured to the output directory for post-engagement review and reporting.

Configuration

Configuration Reference

All configurable parameters available in server.properties and config/app directory.

Server Settings

Key Default Description
server.host 0.0.0.0 Bind address for the C2 listener
server.port 4444 Default listener port
server.timeout 30000 Session timeout in milliseconds
server.backlog 50 Maximum pending connection queue

Output Settings

Key Default Description
output.dir ./output Directory for session logs and downloads
output.log true Enable command and output logging
output.timestamp true Prefix log entries with timestamps

Security Settings

Key Default Description
crypto.enabled true Enable session encryption
crypto.algorithm AES Encryption algorithm for C2 traffic
crypto.keysize 256 Key size in bits for encryption
